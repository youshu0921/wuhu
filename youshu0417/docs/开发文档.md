# 有书相亲小程序开发文档

## 1. 项目概览
- 项目名称：有书相亲（微信小程序）
- 技术栈：原生微信小程序（WXML/WXSS/JS）+ 云函数
- 主要能力：推荐匹配、资料编辑、收藏、消息、广场、每日打卡积分、实名认证

## 2. 目录结构
- `app.js`：小程序启动逻辑（云能力初始化、本地用户 ID）
- `app.json`：页面路由、TabBar
- `utils/storage.js`：本地存储 Key 与读写封装
- `utils/mockData.js`：推荐用户、广场帖子、资料库 mock 数据
- `pages/*`：页面代码
- `cloudfunctions/getUserInfo`：获取 openid 的云函数（可选）
- `cloudfunctions/realnameVerify`：实名认证云函数（对接阿里实名认证 API）

## 3. 页面说明
- `pages/index`：推荐页（换一个、匹配排序、积分与额度展示）
- `pages/mine`：我的页（资料概览、实名认证入口、打卡、收藏入口）
- `pages/profileEdit`：资料编辑页（城市/职业/学历/收入/有车有房等）
- `pages/realname`：实名认证页（姓名+身份证）
- `pages/detail`：用户详情页
- `pages/favorites`：收藏列表页
- `pages/messages`：消息列表页
- `pages/chatRoom`：聊天页
- `pages/square`：广场页
- `pages/library`：资料库页

## 4. 本地存储设计
定义在 `utils/storage.js`：
- `USER_KEY = userInfo`：用户资料
- `POSTS_KEY = posts`：广场动态
- `LIKED_USERS_KEY = likedUsers`：消息列表用户
- `FAVORITES_KEY = favorites`：收藏列表
- `CHECK_IN_KEY = checkInData`：打卡数据
- `POINTS_KEY = pointsData`：积分数据
- `SWAP_USAGE_KEY = swapUsageData`：换一个使用记录

## 5. 核心业务规则

### 5.1 换一个额度与积分
实现文件：`pages/index/index.js`
- 每日免费换人上限：`FREE_SWAP_LIMIT = 20`
- 超出后每次消耗：`PAID_SWAP_COST = 5` 积分
- **重置周期：按天重置（不是按月）**
  - 通过 `getTodayKey()` 生成当日 key（`YYYY-MM-DD`）
  - 在 `getSwapState()` 中，如果本地记录日期不是今天则重置 `count=0`

### 5.2 每日打卡
实现文件：`pages/mine/mine.js`
- 每天首次打卡 +10 积分
- 记录当月打卡日期，用于弹层展示

### 5.3 推荐排序（积分加权）
实现文件：`pages/index/index.js`
- 综合维度：同城、实名、兼容度（身高/体重/收入区间）、车房标签
- 积分越高，推荐权重和容差越高

## 6. 实名认证接入

### 6.1 原因
前端直接请求阿里域名会被微信 request 合法域名拦截。

### 6.2 当前方案（已落地）
- 前端调用云函数：`wx.cloud.callFunction({ name: 'realnameVerify' })`
- 云函数再请求阿里接口：
  - Host: `kzidcardv1.market.alicloudapi.com`
  - Path: `/api/id_card/check`
  - Method: `GET`
  - Header: `Authorization: APPCODE xxx`
  - Query: `name`, `idcard`

### 6.3 相关文件
- 前端页：`pages/realname/*`
- 前端入口：`pages/mine/mine.wxml` + `pages/mine/mine.js`
- 云函数：`cloudfunctions/realnameVerify/index.js`

### 6.4 认证成功后写入字段（USER_KEY）
- `realName`
- `idCardMasked`
- `realNameVerified`（布尔）
- `verifyTime`
- `realNameVerifyMsg`

## 7. 云函数部署步骤
1. 打开微信开发者工具
2. 右键 `cloudfunctions/realnameVerify` -> 上传并部署（安装依赖）
3. 确认云开发环境与项目一致
4. 重新编译小程序

## 8. 常见问题排查

### 8.1 request 合法域名报错
- 现已通过云函数代理规避
- 若仍报错，确认前端不再直接调用阿里域名

### 8.2 实名认证失败
- 检查云函数是否已部署
- 检查阿里接口配额是否充足
- 检查姓名和身份证号格式

### 8.3 页面白屏/点击无响应
- 清缓存并重编译
- 检查控制台首条红色报错
- 优先检查近期改动的 WXML 标签是否完整

## 9. 安全建议（重要）
- 当前云函数中包含第三方 API 凭据（用于快速联调）
- 建议迁移到云环境变量或安全存储，不要长期硬编码在仓库

## 10. 后续建议
- 将实名认证结果落库（云数据库）并做风控日志
- 增加接口失败重试与限流保护
- 将推荐规则参数配置化，便于运营调整
